

You now have all the tools you need to get started on Ansible on your own. It's time to validate your learning through a business problem. If needed, the Ansible documentation can help you go deeper into certain concepts or review those you already know.

Context and purpose:

You have been recruited as a DevOps engineer in a new e-commerce startup that wants to create and deploy a new website.

After consultation with the developer team, it has been determined that the solution used will be Magento for the web server and MySQL to store the data. As a DevOps engineer, your goal is to automate this deployment by creating two roles:

One role will need to deploy Magento.

Another role will need to deploy a MySql database.

It is critical that the web server can communicate with the database.

In order to verify that your roles are performing their tasks as expected, you are responsible for writing unit tests to validate their operation prior to production deployment for the company's e-commerce site. These tests will be used to verify the connection between the web server Magento and the database MySql. So that users have good access to the homepage of the site.

Introduction to the Magento solution

Magento is an e-commerce platform built on open source technology that provides online merchants with a flexible shopping cart system, as well as control over the look, feel and functionality of their online store. Magento offers powerful marketing, search engine optimization and catalog management tools. Magento is for businesses that need an e-commerce site offering many services or products, but are also looking for customization.
